# 系统设计基础

## 核心概念

### 1. 可扩展性 (Scalability)
- **垂直扩展**: 增加单机硬件配置
- **水平扩展**: 增加服务器数量
- **分片**: 数据分散存储

### 2. 可用性 (Availability)
- **冗余**: 多副本部署
- **故障转移**: 主从切换
- **健康检查**: 及时发现故障

### 3. 一致性 (Consistency)
- **强一致性**: 每次读取最新数据
- **最终一致性**: 允许短暂不一致
- **CAP 定理**: 分区容错时，权衡一致性与可用性

---

## 常见架构

### 1. 单体架构
- 简单、适合小项目
- 缺点: 难以扩展、耦合高

### 2. 微服务架构
- 服务拆分、独立部署
- 优点: 独立扩展、技术多样性
- 挑战: 服务治理、分布式事务

### 3. 事件驱动架构
- 基于消息队列解耦
- 优点: 异步、松耦合
- 挑战: 消息顺序、幂等性

## 关键技术

### 1. 负载均衡
| 方式 | 说明 |
|------|------|
| **Round Robin** | 轮询 |
| **Least Connections** | 最少连接 |
| **IP Hash** | 源IP固定 |
| **加权** | 权重分配 |

### 2. 缓存
| 策略 | 说明 |
|------|------|
| **Cache-Aside** | 应用负责缓存 |
| **Read Through** | 缓存负责加载 |
| **Write Through** | 同步写缓存和DB |
| **Write Behind** | 异步写缓存 |

### 3. 数据库
| 类型 | 用途 |
|------|------|
| **关系型 (MySQL)** | 事务、一致性 |
| **NoSQL (MongoDB)** | 灵活、扩展性 |
| **缓存 (Redis)** | 高速访问 |
| **搜索引擎 (ES)** | 全文搜索 |

### 4. 消息队列
| 场景 | 工具 |
|------|------|
| **异步处理** | RabbitMQ, Kafka |
| **任务队列** | Celery |
| **事件流** | Kafka, Pulsar |

## 设计模式

### 1. 分布式
- **一致性哈希**: 数据分片
- **Gossip 协议**: 节点同步
- **Paxos/Raft**: 共识算法

### 2. 高可用
- **主从复制**: 数据冗余
- **熔断器**: 故障隔离
- **限流**: 保护系统

### 3. 性能
- **读写分离**: 分担负载
- **分库分表**: 数据拆分
- **CDN**: 静态资源加速
