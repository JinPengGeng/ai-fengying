# API 设计与开发

## RESTful API

### 原则

| 原则 | 说明 |
|------|------|
| 资源导向 | 使用名词而非动词 |
| 无状态 | 每个请求独立 |
| 统一接口 | 标准 HTTP 方法 |
| 分层系统 | 客户端无需知道后端 |

### HTTP 方法

| 方法 | 用途 | 幂等 |
|------|------|------|
| GET | 获取资源 | ✅ |
| POST | 创建资源 | ❌ |
| PUT | 替换资源 | ✅ |
| PATCH | 部分更新 | ❌ |
| DELETE | 删除资源 | ✅ |

### 状态码

| 状态码 | 含义 |
|--------|------|
| 200 | 成功 |
| 201 | 创建成功 |
| 204 | 无内容 |
| 400 | 请求错误 |
| 401 | 未授权 |
| 403 | 禁止访问 |
| 404 | 未找到 |
| 500 | 服务器错误 |

## GraphQL

### 特点

- 获取需要的数据
- 单次请求多个资源
- 强类型 schema

### 示例

```graphql
# 查询
query {
  user(id: "1") {
    name
    email
    posts {
      title
    }
  }
}

# 变更
mutation {
  createUser(input: {name: "Tom"}) {
    id
    name
  }
}
```

## API 安全

### 认证方式

| 方式 | 说明 |
|------|------|
| API Key | 简单密钥 |
| JWT | 无状态令牌 |
| OAuth 2.0 | 第三方授权 |

### 安全措施

| 措施 | 用途 |
|------|------|
| HTTPS | 加密传输 |
| 限流 | 防止滥用 |
| CORS | 跨域控制 |
| 输入验证 | 防止注入 |
