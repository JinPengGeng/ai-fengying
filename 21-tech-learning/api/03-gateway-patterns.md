# API Gateway 设计模式

## 核心模式

### 1. 请求路由
- 基于路径、Header、Query参数的路由
- 正则表达式匹配
- 动态路由模板

### 2. 认证与授权
- JWT 验证
- API Key 管理
- OAuth 2.0 集成
- mTLS 双向认证

### 3. 限流与熔断
- 令牌桶算法
- 滑动窗口限流
- 熔断器模式
- 降级策略

### 4. 缓存策略
- CDN 集成
- 响应缓存
- 条件请求 (ETag, Last-Modified)

### 5. 监控与日志
- 请求/响应日志
- 性能指标采集
- 分布式追踪
- 告警规则

---

## 常用网关

| 网关 | 特点 |
|------|------|
| Kong | 插件丰富，性能好 |
| Nginx | 高性能，反向代理 |
| Traefik | 云原生，动态配置 |
| APISIX | 高性能，Apache 顶级项目 |
| AWS API Gateway | AWS 生态集成 |
