# Prompt Engineering 进阶指南

## 核心原则

### 1. 清晰明确（Clear）
- 使用具体、明确的语言
- 避免模糊或歧义
- 说明格式要求

### 2. 上下文（Context）
- 提供背景信息
- 说明任务目标
- 给出示例

### 3. 约束（Constraints）
- 明确输出格式
- 设定限制条件
- 说明不要做什么

---

## 高级技巧

### 1. Few-Shot Prompting
```
例子：
输入：我今天很开心
情感：正面

输入：下雨了
情感：负面

输入：考试通过了
情感：？
```

### 2. Chain-of-Thought (CoT)
```
让我们一步步思考：
1. 首先...
2. 然后...
3. 最后...
所以答案是...
```

### 3. ReAct Prompting
```
思考：我需要...
行动：调用搜索工具
观察：结果显示...
思考：现在我需要...
```

### 4. Self-Consistency
- 生成多个答案，选择最一致的

### 5. Tree of Thoughts
- 探索多种思考路径
- 评估每条路径

## 让我变强的 Prompt 模式

### 角色扮演
```
你是一位 [角色]，擅长 [领域]。
请用 [风格] 回答以下问题...
```

### 分解任务
```
任务：[复杂任务]
步骤1：
步骤2：
步骤3：
请按步骤执行。
```

### 反思修正
```
回答这个问题：[问题]
1. 先给出答案
2. 列出你的推理过程
3. 检查可能的错误
4. 修正并给出最终答案
```

---

## 常见模式

| 模式 | 用途 | 示例 |
|------|------|------|
| 角色 | 个性化输出 | "你是一位资深医生" |
| 例子 | 演示格式 | 见 Few-Shot |
| 步骤 | 流程任务 | "按步骤1.2.3." |
| 模板 | 格式化输出 | "用JSON格式返回" |
| 约束 | 限制输出 | "不超过100字" |
