# ReAct 架构深度解析

## 什么是 ReAct？

**ReAct** = Reason（推理） + Act（行动）

是一种让 AI Agent 能够"思考然后行动"的架构模式。

---

## 核心原理

```
输入 → 思考 → 行动 → 观察 → 思考 → 行动 → ... → 输出
```

### 1. 思考 (Thought)
- 分析当前情况
- 决定下一步做什么
- 推理出合理的行动

### 2. 行动 (Action)
- 调用工具
- 搜索信息
- 执行操作

### 3. 观察 (Observation)
- 获取行动结果
- 分析结果
- 决定下一步

---

## 让我变强的关键

### ReAct vs 传统 LLM

| 维度 | 传统 LLM | ReAct Agent |
|------|----------|-------------|
| **单次回复** | ✅ | ❌ |
| **多步骤** | ❌ | ✅ |
| **工具调用** | ❌ | ✅ |
| **自我反思** | ❌ | ✅ |

### 循环机制

```python
# ReAct 伪代码
while not done:
    # 1. 思考
    thought = llm.think(context)
    
    # 2. 决定行动
    action = extract_action(thought)
    
    # 3. 执行行动
    result = execute(action)
    
    # 4. 观察结果
    context += f"行动: {action}\n结果: {result}"
    
    # 5. 检查是否完成
    done = llm.is_done(context)
```

## ReAct 的变体

### 1. ReAct + Tool Use
- 行动 = 调用 API/工具
- 观察 = 工具返回结果

### 2. ReAct + Memory
- 增加记忆机制
- 短期记忆（上下文）
- 长期记忆（向量库）

### 3. ReAct + Planning
- 先生成计划
- 然后逐步执行
- 可调整计划

### 4. ReAct + Self-Correction
- 行动后检查结果
- 如果错误，自我修正
- 重新尝试

## 实际应用

### 1. 搜索引擎 Agent
```
思考: 我需要找到2026年AI的最新动态
行动: 搜索 "AI news 2026"
观察: 返回了结果列表
思考: 第一个结果看起来最相关
行动: 打开第一个链接
观察: 获取了详细内容
...
```

### 2. 编程 Agent
```
思考: 用户想要一个Python脚本
行动: 分析需求，列出步骤
行动: 编写代码
行动: 运行测试
观察: 测试失败
思考: 需要修复错误
行动: 修正代码
...
```

### 3. 数据分析 Agent
```
思考: 用户想要分析销售数据
行动: 读取数据文件
观察: 数据加载成功
行动: 统计基本指标
观察: 获得均值、方差等
行动: 可视化
...
```

## 与其他架构对比

| 架构 | 特点 | 适用场景 |
|------|------|----------|
| **ReAct** | 推理+行动 | 通用任务 |
| **CoT** | 纯推理 | 数学、逻辑 |
| **ToT** | 树形搜索 | 复杂规划 |
| **FoA** | 聚焦-agent | 信息检索 |
